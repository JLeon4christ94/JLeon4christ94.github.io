Variables

Reusing Values in Variables

One reason we use variables is that they allow us to easily reuse values in different parts of our code.

When we reuse a value, it will appear in multiple places in our code. Re-typing that value becomes tedious, which can lead to errors. Without a variable name, it may also be unclear what the value represents.

Let’s take a look at the following piece of code. Here, we have a number that we reuse to make some calculations:

847595593392818109495
847595593392818109495 * 2
847595593392818109495 / 4

Rather than writing the same number over and over again, we can save it to a variable named my_number: 

my_number = 847595593392818109495
my_number * 2
my_number / 4

You may be thinking, “But what if my variable name is longer than the value it stores? What’s the point of a variable?”

Using a value without assigning it to a variable is known as hardcoding. While it’s sometimes faster to initially hardcode values in your program, in the long run, you’ll run into trouble — especially if you need to change what those values are.
